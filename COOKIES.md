# Cookie Management - Paste Netscape Format

## Overview
You can now paste Netscape format cookies directly into the web UI instead of just providing a file path. The pasted cookies are automatically saved to `data/cookies.txt`.

## How to Use

### Method 1: Paste Cookies via Web UI
1. Go to the Configuration tab
2. Find the "Or Paste Cookies (Netscape Format)" section
3. Paste your Netscape format cookies into the textarea
4. Click "Save Pasted Cookies" button
5. The cookies are saved and the configuration is updated automatically

### Method 2: Browser Cookies (Existing)
1. Select your browser from "Browser Cookie Source" dropdown
2. Requires browser to be running and logged in to YouTube
3. Save Configuration

## Cookie Format (Netscape)

The expected format is:
```
# Netscape HTTP Cookie File
# This file is generated by yt-dlp.
.youtube.com	TRUE	/	TRUE	1805237469	COOKIE_NAME	cookie_value
.youtube.com	TRUE	/	TRUE	1805237469	ANOTHER_COOKIE	another_value
```

Columns: domain, flag, path, secure, expiry, name, value

## API Endpoint

### POST /api/cookies
Save Netscape format cookies.

Request body:
```json
{
  "cookie_text": "# Netscape HTTP Cookie File\n.youtube.com\tTRUE\t/\tTRUE\t1805237469\tYSC\tvalue123"
}
```

Response:
```json
{
  "success": true,
  "data": {
    "message": "Cookies saved successfully"
  }
}
```

## What Happens Behind the Scenes

1. Cookies are validated as plain text
2. Saved to `data/cookies.txt`
3. Configuration updated to use `cookies_file: ./cookies.txt`
4. `cookies_browser` cleared (so pasted cookies take precedence)
5. Changes persist across restarts

## File Location

The cookies file is stored at: `data/cookies.txt`

This is part of the data directory that's mounted in Docker:
```yaml
volumes:
  - ./data:/app/data
```

Both local and Docker deployments use the same file location and format.
